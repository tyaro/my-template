## 3. 保留中: PR-C（Tauri CSP）— 再開に必要な材料

`apps/admin-template/src-tauri/tauri.conf.json` の `security.csp` が現在 `null`。設定すべきだが、
**厳格すぎる CSP はアプリを静かに壊す**ため、実機ビルド+スモークでの反復が必須
（オーナーが受け入れ条件に「Tauri 実動スモーク」を指定）。このサンドボックスでは
`src-tauri` をビルド/実行できないため、値を確定できない。

**監査で判明した CSP 上の制約**:

- `app.html` に**インライン `<script>`**（テーマちらつき防止 IIFE）→ `script-src` に hash か
  `'unsafe-inline'` が必要（Tauri の nonce 自動付与は Tauri 注入スクリプトのみで著者インラインは対象外）。
- Svelte のランタイム `<style>` 注入 → `style-src 'unsafe-inline'` がほぼ必須。
- 添付/バックアップ/CSV/チャート出力の `URL.createObjectURL`（blob:）→ `img-src blob:` 等。
- ログイン背景の `data:image/svg+xml` → `img-src data:`。
- Tauri `invoke()` → `connect-src 'self' ipc: http://ipc.localhost`。

**提案する叩き台 CSP**（実機で締め上げる前提）:

```
default-src 'self';
script-src 'self' 'sha256-<theme-script のハッシュ>';   // 暫定は 'unsafe-inline'。hash 値は実ビルド出力で確定
style-src 'self' 'unsafe-inline';
img-src 'self' data: blob:;
font-src 'self';
connect-src 'self' ipc: http://ipc.localhost;
object-src 'none'; base-uri 'self'; frame-ancestors 'none'; form-action 'self'
```

補足: tauri.conf の CSP は**デスクトップ Webview のみ**に効く（LAN ブラウザ配信側は
`banto-server` のセキュリティヘッダが別途担う）。インラインスクリプトの hash 化は
SvelteKit `kit.csp`（hash モードで app.html も自動ハッシュ）に寄せる案も有力 — これも実機確認要。

**実動スモーク・チェックリスト（PR-C の受け入れ条件）**: CSP 適用後、実機で ① ログイン/初回
セットアップ ② `invoke()`（items 一覧・作成）③ テーマ切替のちらつき無し ④ 添付の画像
プレビュー/サムネイル（blob:）⑤ CSV/バックアップのダウンロード ⑥ 設定画面の QR 表示
⑦ Console に CSP 違反ゼロ、が壊れないこと。

**再開手順**: 実機で `claude/p0-tauri-csp` を `main` から作成 → 叩き台を適用 → チェックリストを回して
値を確定 → PR 作成。

## 6. 環境と検証コマンド（このリポジトリの制約）

- **`src-tauri` はこのサンドボックスでコンパイル不可**（webkit2gtk 不在）。Tauri コマンド側の
  変更はコードレビュー + 静的検査（`verify:architecture` rule 8 / CR-6）+ `tauri-check.yml`
  （週次/パス条件付き）で担保する。
- 検証: `pnpm check`（型）/ `pnpm verify:architecture`（9ルール+CR-6）/ `pnpm check:versions`
  （CR-7、PR-B ブランチ以降）/ `cargo test -p admin-template-core -p banto-server ...` /
  `node --test scripts/rename.test.mjs`（AD-5 ブランチ）/ `prettier --check` / `eslint`。
- **依存を足さない**（conventions §3）: 追加スクリプト（verify-architecture / check-versions /
  rename.test）はすべて Node 標準のみ。
- ブランチ規律: 開発は指定ブランチで、他ブランチへの push は明示許可が要る運用だった
  （オーナーが「PR を分ける」と指示済み。P0 は `claude/p0-*`）。
